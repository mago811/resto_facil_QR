Sección 3 — Flujos clave

Flujo del cliente (QR → factura) y flujos del panel admin

Flujo 1 — Cliente (público)
1. Escanea QR
URL: /factura?mesa=12
RSC carga la sesión POS activa de la mesa.
Si no hay sesión → error 404 "Mesa sin consumo activo"
2. Ve el ticket
RSC muestra resumen del consumo: subtotal, IVA 10%, total.
Datos del restaurante emisor visibles.
3. Formulario fiscal
"use client" solo en inputs.
Campos: tipo doc · NIF/CIF/NIE · Razón social · Dirección
Validación Módulo 11 on-blur (sin round-trip).
4. Server Action: emitInvoice()
Zod valida payload completo.
Crea fila en facturas. Genera PDF con react-pdf. Sube a Vercel Blob. Actualiza pdf_url. Marca sesión como facturada.
Transacción Drizzle — todo o nada.
5. Confirmación
Página de éxito con botón "Descargar PDF".
Link directo al Blob URL. Hook ESC/POS disponible aquí para cuando haya impresora real.
Flujo 2 — Panel Admin (auth requerida)
Mesas + QR
  1. Crear/editar mesa (número, nombre)
  2. Generar QR → imagen PNG con URL /factura?mesa=N
  3. Crear sesión POS mock (subtotal, descripción)
  4. Ver estado: abierta / facturada / cerrada
Facturas
  1. Listado paginado con filtros (fecha, mesa, NIF)
  2. Ver detalle de factura
  3. Re-descargar PDF
  4. Exportar CSV (AEAT)
Configuración
  1. Datos fiscales del restaurante (CIF, razón social, dir.)
  2. IVA por defecto (10% hostelería, editable)
  3. Gestión de usuarios admin
  4. Placeholder webhook ESC/POS futuro
Validación NIF/CIF/NIE (Módulo 11) — implementada en shared/lib/validators/spanish-id.ts, pura TS sin dependencias. Se ejecuta en el cliente (on-blur) para feedback inmediato y se re-valida en el Server Action antes de persistir.
